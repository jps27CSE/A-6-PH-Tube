Discuss the scope of var, let, and const

Scope of Variables

In JavaScript, the scope of variables can be categorized into three types:

var:

Variables declared with var are function-scoped. They are accessible throughout the entire function in which they are declared, regardless of block scope.

let:

Variables declared with let are block-scoped. They are accessible only within the block (enclosed by curly braces) where they are defined.

const:

Variables declared with const are also block-scoped like let. However, they cannot be reassigned after declaration, making them suitable for constants.

Tell us the use cases of null and undefined

Use Cases of null and undefined

null:

null is a deliberate assignment value that represents the absence of any object value. It's typically used when you want to indicate that a variable or object property intentionally has no value or doesn't point to any object.

Example use cases of null:

  • Initializing a variable that will later hold an object.
  • Clearing the value of an object property to indicate it's empty.

undefined:

undefined represents a variable that has been declared but has not been assigned a value. It is the default value of uninitialized variables and function parameters.

Example use cases of undefined:

  • Variables that haven't been assigned a value yet.
  • Accessing an object property that doesn't exist.
  • Function parameters that were not provided when the function is called.

What is a REST API?

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services over the HTTP protocol.

Key characteristics of RESTful APIs include:

  • Using standard HTTP methods (GET, POST, PUT, DELETE) for CRUD operations.
  • Resources represented as URLs.
  • Statelessness, where each request contains all necessary information.
  • Common data exchange formats like JSON or XML.

REST APIs are widely used for building web applications and enabling communication between different services and clients.
